## What we did, roughly in order

1. **Three-way inventory.** Read the Paper "design tokens" board (artboard `1-0`, "CANONICAL :root SYSTEM — DARK THEME"), `web/src/app/globals.css`, and grepped the component tree for hardcoded color/font literals. The board's token registry is empty — tokens live as a visual spec, which matched globals.css value-for-value except the board had dropped `--secondary-foreground` and `--accent-foreground`.

2. **Reconciled blueprint.** Collapsed the flat token groups into five named ramps — Grounds (neutral), Steel (cool blue-grey chrome), Forest (greens), Amber (warm brand), Type (ink) — numbered 50→950 by measured relative luminance. Category palette stays its own 8×2 dimension (with `--cat-fuel` red and `--cat-oddity` purple as intentional outliers). Borders/detail stay literal alpha overlays.

3. **Precondition verifications.** Resolved three open questions before generating (see Decisions). The big one: Steel had mixed neutral greys with saturated interactive blues; we pulled the blues out into named roles so the ramp is a clean grey progression.

4. **Generated `DESIGN.md`.** Authority header, primitive block, full semantic-alias mapping, the generated `--color-*` mirror documented, usage rules, component conventions, do/don't drift-killers, and the sync-topology note.

5. **Aligned `globals.css`.** Rewrote `:root` to: ramp primitives (editable values) → semantic aliases (every legacy name repointed) → added scales. Marked `@theme inline` as a generated mirror and added `--color-*` for the new ramps/roles + `--font-display-condensed`. Everything below `:root` (`.form-field`, `.section-label`, `.rdp-themed`, scrollbars) untouched.

6. **Enforcement + verification.** Added a Styling rule to `web/AGENTS.md`. Static check confirmed: 79/79 legacy aliases preserved, brace balance OK, no real dangling vars (only the runtime `--font-*` from next/font), and every spot-checked alias resolves to its exact pre-refactor value.

## Decisions made today

- **Five ramps as raw primitives + semantic aliases.** The ramp step holds the value; legacy names (`--bg-*`, `--input-*`, `--button-*`) become `var(--ramp-step)` aliases. Keeps all existing references working while giving one canonical scale. No alias renamed.
- **Source-of-truth topology.** `globals.css :root` primitives = the only editable value source → `--color-*` mirror is generated from them → `DESIGN.md` is the single generated reference everyone reads → Paper is the visual surface, reconciled into globals.css on demand. Artboards use `var(--token)` only, never raw hex.
- **Pin/marker = `#FF8E05`** (verification 1). The orange-red was untokenized and drifted across `#FF8E05 · #F88112 · #F68A0D · #EF8B23 · #E8941F`; canonicalized to the ranking-glyph value. **Not** reassigned to amber — pins are orange-red, amber is a text/data/active accent only.
- **Space Grotesk is real** (verification 2), 41 usages via `--ff-display`. The "maybe it's Space Mono + Barlow + Barlow Condensed" hypothesis was false.
- **Barlow Condensed tokenized.** Loaded in `layout.tsx` and used for card titles but had no token → added `--ff-display-condensed`. **Inter** is in Paper's font list but loaded/used nowhere → stray, not encoded.
- **Crimson Text kept.** Loaded AND used (`trip-detail-header.tsx:118` editorial kicker, `category-planning-slide.tsx:42`); already tokenized as `--ff-serif`. Not stray — documented as-is.
- **Added scales marked "not from Paper":** spacing base-4, `--radius-full`, border-width 1/2, shadows sm/md/lg/inset, full type scale (sizes/weights/leading/tracking), `--warning`.

## Hard-won details — gotchas worth not rediscovering

- **Paper token registry is empty** (`get_tokens` → `[]`) for this file — the tokens are a *visual spec* drawn on artboard `1-0`, not Paper variables. Read them via `get_jsx` / screenshot, not `get_tokens`.
- **`get_jsx` on a full artboard overflows** the tool result (~62k chars). Dump-to-file + grep, or hand to a subagent in chunks.
- **next/font vars look "dangling" in CSS.** `--font-barlow`, `--font-space-grotesk`, etc. are injected at runtime by `next/font` via the `className` in `layout.tsx` — they're correctly undefined inside globals.css. Don't "fix" them.
- **The diary template is `Diary/design_tempalte.md`** (misspelled), not `TEMPLATE.md`.
- **Pure-indirection refactors need value verification, not a browser.** A token-aliasing change has zero visual delta by construction; verify by chasing each `var()` chain to its literal and diffing against the original, not by screenshotting.
